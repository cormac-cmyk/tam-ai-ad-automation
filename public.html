Overview

What This Automation Does

This workflow uses Manus AI to automate the process of creating Meta ads. You give Manus a Google Drive folder containing your creative assets (videos or images), and it handles the rest — downloading the files, uploading them to Meta, generating ad copy, and creating the ads in your chosen campaign. All ads are created in a PAUSED state so you can review them before anything goes live.

This is not a plug-and-play tool. It requires a one-time technical setup involving two third-party platforms: the Meta Developer Portal and the Google Cloud Console. Both are free to use. The setup takes most people between 1–3 hours on their first attempt.

⚠ Real Risk — Ad Account Bans Using the Meta Marketing API without a properly configured Developer App is one of the most common reasons ad accounts get restricted or banned. Do not skip the setup steps. Do not share your System User token with anyone. TAM is not liable for any account restrictions that occur as a result of using this automation.

Setup — Part 1 of 3

Meta Developer App & System User Token

This is the most important part of the setup. You are creating a registered application inside Meta's developer platform, then generating a permanent token that Manus will use to create ads on your behalf.

⚠ Before You Start You must have a Meta Business Manager account and an active ad account before completing these steps. If you do not have these, set them up first at business.facebook.com.
1
Meta Developer Portal

Create a Developer App

Go to developers.facebook.com → click My Apps → Create App. Select Business as the app type. Give it a name (e.g. "TAM Ad Automation"). Connect it to your Business Manager account.

2
Meta Developer Portal

Add the Marketing API Use Case

Inside your new app, go to Use Cases → find Advertise → click Set Up. This adds the Marketing API permissions your app needs to create ads.

3
Meta Business Manager

Create a System User

Go to business.facebook.com → Business Settings → Users → System Users → click Add. Name it (e.g. "Manus System User") and set the role to Admin.

4
Meta Business Manager

Assign Your Ad Account to the System User

Still in System Users — click your new System User → Add Assets → select Ad Accounts → choose your ad account → grant Full Control.

5
Meta Business Manager

Generate a System User Token

Click Generate New Token on your System User → select your Developer App → tick the ads_management and ads_read permissions → click Generate Token. Copy and save this token somewhere secure — you will not be able to see it again.

⚠ Disclaimer — Token Security Your System User token grants full access to create and modify ads on your ad account. Never share it publicly, paste it into a chat, or store it in an unsecured location. TAM is not responsible for any unauthorised use of your token or any resulting account actions by Meta.
🤖
Getting lost in Meta's interface? Meta's Business Manager UI changes regularly. If any of the above steps don't match what you're seeing on screen, take a screenshot and ask Manus: "I'm trying to create a System User in Meta Business Manager — this is what I'm seeing. What should I click next?"
Setup — Part 2 of 3

Google Cloud Console — Drive API & OAuth

This part allows Manus to access your Google Drive folder to download your creative assets. You are enabling the Drive API and creating a set of credentials (a Client ID and Client Secret) that Manus will use to authenticate.

1
Google Cloud Console

Create a New Project

Go to console.cloud.google.com. At the top of the page, click the project selector dropdown → click New Project → give it a name (e.g. "TAM Ad Automation") → click Create.

2
Google Cloud Console

Enable the Google Drive API

From your project dashboard, click the APIs and services tile → click Enable APIs and Services → search for Google Drive API → click it → click Enable.

3
Google Cloud Console

Configure the OAuth Consent Screen

Go to Credentials in the left menu. You will see a yellow banner prompting you to configure the consent screen — click it. Work through the 4-step wizard: set the app name to "TAM Ad Automation", add your email, and complete the steps. Set the audience to External.

4
Google Cloud Console

Create OAuth Credentials

Back on the Credentials page → click Create Credentials → select OAuth client ID → choose Desktop app as the application type → name it "Manus Desktop Client" → click Create. Download the JSON file — you will need the Client ID and Client Secret from it.

🤖
Google Cloud's interface can be confusing. If you can't find a button or the page looks different to what's described above, take a screenshot and upload it to Manus with the message: "I'm setting up Google OAuth credentials for the Meta Ads automation — I'm stuck here. What do I do next?"
Setup — Part 3 of 3

Setting Up the Skill in Manus

Once you have your Meta System User token and your Google OAuth credentials, the final step is configuring the skill inside Manus so it can use them.

1
Manus AI

Start a New Task in Manus

Open manus.im and start a new task. Tell Manus: "I want to set up the Meta Ads automation skill. I have my System User token and my Google OAuth credentials ready." Manus will walk you through the rest of the configuration interactively.

2
Manus AI

Provide Your Credentials When Asked

Manus will ask you for your Meta System User token, your Ad Account ID, your Facebook Page ID, your Google Client ID, and your Google Client Secret. Have these ready to paste in when prompted.

3
Manus AI

Run Your First Test

Once setup is complete, test the skill by providing a Google Drive folder link containing a single creative. Tell Manus which campaign to use and confirm the destination URL. Review the PAUSED ad in Meta Ads Manager before activating anything.

⚠ Start Small On your first run, use a single creative in a test campaign. Do not attempt to upload 20+ creatives across multiple ad accounts until you have confirmed the skill is working correctly on your own account. Running high volumes of API calls too quickly is a known trigger for Meta account flags.
Know the Risks

What Can Go Wrong

This is not a scare tactic — these are real risks that you need to understand before you begin. TAM has documented these based on how Meta's systems work and the most common failure points reported by users of API-based automation.

!

Ad Account Bans

Meta can restrict or permanently ban ad accounts that it detects are being used in ways that violate its policies. Automated ad creation via the API is permitted — but only when done through a properly registered Developer App. Accounts using unofficial methods, shared tokens, or unusually high API call volumes are at higher risk.

!

Token Compromise

If your System User token is exposed or shared, a third party could use it to create, modify, or delete ads on your account. Always store your token securely and regenerate it immediately if you believe it has been compromised.

!

Policy Changes

Meta updates its API policies, rate limits, and advertising rules regularly. This guide reflects the setup as of the date it was published. TAM cannot guarantee that these steps will remain accurate or that this automation will remain compliant with future Meta policy changes.
